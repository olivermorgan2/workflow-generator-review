# Skills audit — 2026-05-07

A one-shot audit harness for the SKILL.md cohort: deterministic
metrics + qualitative rubric + independent third-party verification.
Fed `notes/refactoring-ideas.md` entry #2 (issue #83 — type-3 ADR
attribution prune; shipped in PR #85) and entry #9 (issue #84 — full
cohort cleanup; Phase 1 shipped in PR #86, Phase 2 + Phase 3 pending).

## What's tracked vs not

The harness directory mixes reusable scripts, curated deliverables,
auto-generated outputs, and superseded v1 drafts. Only the reusable
and curated subset is committed:

**Tracked (this directory):**

- `audit.py` — deterministic SKILL.md metrics. Reusable across audits.
  Re-run before each phase ships to confirm the cohort still meets
  budget rules.
- `adr-audit.py` — type-3 ADR-attribution detector. Issue-#83 specific
  but kept as a worked example for future noise-bucket audits.
- `verify.py` — sends the audit deliverables to OpenRouter for
  independent verification. Model-agnostic; takes a model id as arg.
  Requires `OPENROUTER_API_KEY` in env.
- `methodology-v2.md` — the current scoring rubric. Cited from
  PR #86. v1 (`methodology.md`) is local-only and preserved as
  audit trail.
- `findings-v2.md` — the current findings document. Cited from
  PR #86. Drives the issue-#84 phase plan.
- `verification-openai_gpt-5.5.md` — curated rendering of the
  third-party verification pass.
- `adr-attributions.md` — manual classification of `adr-audit.py`'s
  raw matches. Companion to issue #83.
- `skills-audit-judgment-v2.csv` — already on main from PR #86. The
  per-skill rubric scores for the v2 methodology.

**Not tracked (kept local):**

- `methodology.md`, `findings.md`, `skills-audit-judgment.csv` — v1
  drafts, explicitly superseded by v2 (see `findings-v2.md` §"What
  changed from v1"). Useful as audit-trail context locally; tracking
  them on the public record would invite confusion.
- `skills-audit.csv` — auto-generated by `audit.py`. Re-run the
  script to regenerate. Tracking would introduce drift between the
  CSV and what the script would produce against the live cohort.
- `adr-attributions.csv` — auto-generated by `adr-audit.py`. Same
  reasoning.
- `verification-openai_gpt-5.5.json`, `verification-openai_gpt-5.5-pro.json`
  — raw OpenRouter API responses kept locally for cost / debugging
  per `verify.py`'s docstring. The curated `.md` is the deliverable.

The `.gitignore` codifies the regenerable patterns
(`skills-audit-*/skills-audit.csv`, `skills-audit-*/adr-attributions.csv`,
`skills-audit-*/verification-*.json`), so future audit runs of the
form `notes/skills-audit-YYYY-MM-DD/` automatically follow the same
convention without per-directory tooling.

## Reproducing the audit

```bash
# 1. Deterministic metrics → notes/skills-audit-2026-05-07/skills-audit.csv
python3 notes/skills-audit-2026-05-07/audit.py

# 2. Type-3 ADR attribution scan (issue #83 harness)
python3 notes/skills-audit-2026-05-07/adr-audit.py

# 3. Independent verification via OpenRouter (rubric pass)
OPENROUTER_API_KEY=... python3 notes/skills-audit-2026-05-07/verify.py openai/gpt-5.5

# 4. Human-judged rubric pass — empty output means all 19 rows clean
awk -F, 'NR>1 && ($2!="yes" || $3!="yes" || $4!="yes" || $5!="yes")' \
  notes/skills-audit-2026-05-07/skills-audit-judgment-v2.csv
```

## Convention for future audits

Each audit run lives in its own dated directory: `notes/skills-audit-YYYY-MM-DD/`.

When the next audit lands, copy this README's "What's tracked vs not"
section as the starting convention; adapt as the audit's deliverables
require. The split between **reusable scripts + curated docs**
(tracked) and **regenerable outputs + raw API responses + superseded
drafts** (untracked) generalises.
